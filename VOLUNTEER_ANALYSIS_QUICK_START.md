# 同工分析功能 - 5分钟快速开始

## 🚀 三句话了解新功能

1. **下周日谁服侍？** → 说 "下周日有哪些同工服侍？"
2. **谁在多个岗位服侍？** → 说 "分析最近一个月同工的岗位分布"
3. **谁服侍太多或太少？** → 说 "分析同工的服侍频率"
4. **未来排班有缺口吗？** → 说 "检查未来4周的排班完整性"
5. **生成主日预览** → 说 "生成下周日的主日预览报告"
6. **岗位人手够吗？** → 说 "分析岗位健康度，找出单点故障"

## 📋 九个功能（3个原有 + 6个新增）

### 原有功能
| 功能 | 提示词 | 一句话说明 |
|------|--------|-----------|
| 查看下周日安排 | `analyze_next_sunday_volunteers` | 自动计算下周日并显示所有服侍安排 |
| 岗位分布分析 | `analyze_recent_volunteer_roles` | 了解同工在哪些岗位服侍，识别多面手 |
| 服侍频率分析 | `analyze_volunteer_frequency` | 找出服侍过多或过少的同工 |

### 新增规划功能
| 功能 | 提示词 | 一句话说明 |
|------|--------|-----------|
| 未来排班检查 | `check_upcoming_schedule` | 检查未来N周排班完整性，识别空缺岗位 |
| 主日预览生成 | `generate_sunday_preview` | 生成指定日期的主日预览报告（证道+同工） |
| 岗位健康度分析 | `analyze_role_health` | 分析岗位健康度，识别单点故障岗位 |
| 讲员轮换分析 | `analyze_preacher_schedule` | 分析讲员轮换模式，建议未来安排 |
| 证道系列追踪 | `track_sermon_series` | 追踪证道系列进度，识别中断 |
| 同工趋势分析 | `analyze_volunteer_growth` | 分析同工趋势，识别新/活跃/休眠同工 |

## 📊 能得到什么信息？

### analyze_next_sunday_volunteers
- ✅ 所有岗位的同工名单
- ✅ 服侍人数统计
- ✅ 空缺岗位提醒

### analyze_recent_volunteer_roles
- ✅ 每位同工的服侍记录
- ✅ 多面手同工识别
- ✅ 岗位轮换分析
- ✅ 培训和优化建议

### analyze_volunteer_frequency
- ✅ 每人服侍次数和频率
- ✅ 服侍过多的同工（需关怀）
- ✅ 服侍较少的同工（可增加）
- ✅ 按岗位的人手分析
- ✅ 排班均衡性评估
